# Concept 04: Event Handling in React

### Core Learnings:
- **camelCase Events**: React uses camelCase naming convention for events (e.g., `onClick`, `onChange`).
- **Function Reference**: Passed functions to events as references (`onClick={handler}`) without parentheses, preventing immediate execution on render.
- **Event Object (`e`)**: Handled user keystrokes in inputs using `e.target.value` to capture inputs dynamically.
- **Form/Input updates**: Linked custom operators to individual click events.

### Project:
- **Simple Calculator**: Built an interactive calculator practicing state management alongside input-change and button-click events.
